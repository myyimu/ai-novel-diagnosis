# 前端 UX 重构：信息架构

日期：2026-07-10

## 信息架构判断

**不建议把永久三栏作为所有功能页面的通用结构。**

这个产品的核心是有明确先后关系的诊断任务，而不是同时监控多个对象的 IDE 或数据后台。三栏可以作为少数“主对象 + 选中详情”场景的桌面增强，但不应成为页面默认信息架构。

推荐采用：

```text
全局工作区导航
→ 当前工作区的任务导航
→ 单一主工作区
→ 仅在选中对象且有补充详情时出现的上下文检查器
```

主工作区始终是视觉焦点。左侧只回答“我在哪个任务”；右侧只回答“我当前选中的这个对象有哪些详情”。

## 核心用户心智模型

### 新手主路径

```text
贴入章节
→ 找到最大劝退点
→ 复制下一版 Prompt
→ 改稿
→ 再次诊断
→ 对比是否变好
```

### 进阶质检路径

```text
准备参考样本
→ 生成评分标准
→ 对待诊章节评分
→ 查看总分、问题和证据
→ 回到改稿/复诊
```

### 整书研究路径

```text
上传整书
→ 检查章节切分
→ 启动异步拆解
→ 查看任务进度
→ 浏览人物/关系/伏笔
→ 对比样本或导出
```

这些路径是顺序型任务。导航、Tab、步骤和详情面板必须服从顺序，而不是把所有阶段同时铺开。

## 内容分类

| 分类 | 当前内容 | 判断与目标位置 |
| --- | --- | --- |
| 主要任务 | 快速诊断；深度质检；整书拆解；样本对比 | 独立路由或路由内明确步骤；主工作区只显示一个任务 |
| 高频操作 | 粘贴章节、用示例、运行/重跑、复制 Prompt、复诊 | 靠近主内容；不放右栏；结果态保留明确主 CTA |
| 次要操作 | 导入参考、调整平台策略、保存备注、导出当前结果 | 当前任务内次级按钮、折叠区或溢出菜单 |
| 当前上下文 | 当前项目、当前模型、当前稿件/任务状态 | 页面标题附近的紧凑上下文条；项目可切换，模型只显示状态与入口 |
| 详情或预览 | 单条证据全文、历史诊断详情、方法论卡详情、图谱节点、章节切分预览 | 桌面上下文检查器；平板抽屉；移动端底部 sheet 或独立详情页 |
| 全局设置 | AI 提供商、主题 | `/settings/provider` 等独立设置页；不常驻主任务右栏 |
| 低频设置 | Base URL、API Key、模型列表、AI 自检、完整平台/流量参数 | 设置页或任务内“高级设置”折叠区，默认关闭 |
| 可延迟展示 | 诊断原理、学习路线、研究就绪度、方法论统计、完整历史 | 首次任务后再提示；帮助或独立页面，不占首屏 |
| 独立页面 | AI 设置、复诊历史、方法论库、诊断看板、整书任务历史、研究资料 | 有独立目标、可分享/回访价值，保留路由 |
| Tab | 同一次诊断报告的概览/证据；同一整书结果的人物/关系/时间线 | 仅用于同一对象的平级视角，不跨任务 |
| 步骤 | 深度质检的参考→标准→评分；整书的上传→预览→运行 | 有前置依赖时用 stepper，而不是平级导航 |
| 折叠区 | 题材与稿件来源高级约束、流量数据、AI 自检、帮助说明 | 默认收起，展开不离开当前任务 |
| 抽屉/检查器 | 单条证据、单次历史、方法论卡、图谱节点 | 仅在用户选中对象时出现，可关闭并恢复焦点 |

## 目标全局导航

### 诊断

- 快速诊断：新手主入口。
- 深度质检：用参考样本建立标准并评分。
- 诊断报告/证据：不是永远可见的一级任务；在有结果时作为当前诊断的子视图或可分享结果 URL。

### 项目

- 复诊记录。
- 改稿规则（当前“方法论库”）。
- 诊断看板。
- 项目导出。

“当前项目”首先是上下文选择器，不应占一个与复诊记录并列的空页面；如果需要项目概览，则其页面必须展示真正的项目摘要，而不是复用快速诊断或 ResultsTab。

### 研究

- 整书拆解。
- 样本对比。
- 套路/规律。
- 研究资料。
- 整书历史任务作为整书拆解的任务列表或子页面，而不是设置项。

### 设置

- AI 提供商。
- 主题/显示（若后续确有需求）。

诊断看板属于项目/诊断结果，不属于设置；历史任务属于整书研究，不属于设置。

## 路由、Tab、步骤和详情的放置建议

| 当前入口 | 目标形态 | 说明 |
| --- | --- | --- |
| `/diagnose/quick` | 独立任务页 | compose / running / report 状态；报告出现后输入折叠为摘要 |
| `/diagnose/deep` | 独立任务页 + stepper | 参考样本 → 评分标准 → 章节评分 |
| `/diagnose/score` | 当前诊断的报告子视图或兼容重定向 | 不再与 deep 并列显示相同内容 |
| `/diagnose/evidence` | 报告内“证据”Tab或带 run/session id 的子路由 | 只有有结果时可用 |
| `/project/current` | 项目概览或兼容入口 | 首选将项目选择移入全局上下文条 |
| `/project/revisions` | 独立列表/详情页 | 选中某次复诊时打开检查器或详情路由 |
| `/project/methodology` | 独立页面 | 搜索、筛选、详情检查器 |
| `/project/export` | 项目内次要页面/动作 | 依赖项目已有资产 |
| `/settings/dashboard` | 迁移到项目或诊断 | 保留旧路径重定向 |
| `/settings/history` | 迁移到研究/整书任务 | 保留旧路径重定向 |
| `/research/book` | 独立任务页 + stepper | 上传 → 预览 → 任务 → 结果 |
| `/research/compare` | 独立研究任务页 | 依赖至少两个可比较样本 |
| `/research/patterns` | 独立浏览页 | 不是所有页面的帮助右栏 |
| `/research/materials` | 独立资料页 | 管理输入资产 |
| `/settings/provider` | 独立设置页 | 不再在“输入”Tab或右栏重复完整设置 |

## 条件式三区域规则

如果保留三栏能力，只能按以下职责执行。

### 左栏的唯一职责

**当前工作区的任务导航。**

- 只展示当前工作区相关入口。
- 选中态必须与 URL 一致。
- 不显示项目详情、帮助、状态、设置表单或结果内容。
- 桌面可折叠，但宽度不需要用户拖拽；固定设计 token 可降低状态和测试复杂度。

### 中栏的唯一职责

**完成当前路由对应的主要任务。**

- 页面只保留一个 H1 和一个主 CTA。
- 任务有依赖时用步骤，不把步骤伪装成四个通用 Tab。
- 结果产生后，主区转为报告优先；原输入折叠为“本次输入摘要”。
- 主区必须独占移动端宽度。

### 右栏的唯一职责

**当前选中对象的补充详情/预览。**

允许的例子：选中一条证据、一次复诊、一张方法论卡、一个图谱节点或一个章节预览。

不允许：全局设置、跨工作区导航、新手教程、永久历史列表、与当前任务无关的项目管理。

### 显示、隐藏与折叠条件

- 默认隐藏。
- 只有用户显式选中可查看详情的对象时显示。
- 对象被删除、路由切换或任务重置时关闭。
- 空状态不打开右栏。
- 桌面可保持上次“开/关”，但选中对象必须按路由/任务隔离；不可用一个全局 key 跨工作区复用。
- 关闭后焦点回到触发它的控件。

### 主要视觉焦点

中栏必须占可用宽度的主要部分。建议桌面：

- 左栏 224–256px；
- 主区最小 640px，常规内容宽度 720–960px；
- 右侧检查器 320–400px，仅按需出现；
- 当无法同时满足主区最小宽度时，右栏自动降级为 overlay，而不是继续挤压中栏。

## 响应式降级

| 视口 | 左导航 | 主工作区 | 详情 |
| --- | --- | --- | --- |
| ≥1280px | 固定/可折叠左栏 | 单一主区 | 条件式右侧检查器 |
| 768–1279px | 折叠导航 rail、顶部菜单或侧边 drawer | 占满剩余宽度 | modal drawer/overlay，不与主区并排 |
| <768px | 顶部标题 + 菜单 drawer；工作区入口可水平滚动但不双层常驻 | 单列、全宽、自然页面滚动 | bottom sheet 或独立详情页 |

额外规则：

- 移动端不提供“三栏/经典”布局切换；只有一种受支持布局。
- 不允许固定 `h-[calc(...)] + overflow-hidden` 把页面切成多个独立滚动容器。
- 触控端不展示拖拽条；所有宽度调整能力必须有键盘替代，否则直接取消。
- 200% 缩放时应按更窄断点处理。

## 状态归属

### 继续保留为业务状态

- 项目、提供商、章节输入、参考文本、Rubric、评分、快速诊断结果；
- 整书上传/job/result；
- 复诊、方法论、研究资产与缓存。

数据结构和含义不在本轮 UX 重构中改变。

### 应从业务 store 分离的展示状态

- 左导航是否展开；
- 当前任务内选中的详情对象；
- 折叠区开关；
- 临时 Tab/step（若不能由 URL 或业务状态推导）。

展示状态应由路由、页面局部状态或按工作区命名的 UI store 管理，不能继续与业务数据使用不分域的全局 key。

### 应优先由 URL 表达

- 当前工作区/任务；
- 可分享的报告或历史详情；
- 同一报告的 summary/evidence 等稳定子视图（如果产品确认需要可分享）。

## 关键设计原则

1. 一屏只回答一个当前问题。
2. 导航表示用户任务；步骤表示前置依赖；Tab 表示同一对象的平级视角。
3. 上下文可见但不抢主任务。
4. 结果出现后，结果优先于输入。
5. 低频设置不常驻，高级能力按需展开。
6. 同一内容只能有一个主要归属；其他入口只能链接过去，不能复制实现。
