# 前端 UX 重构：验收标准

日期：2026-07-10

## 总体验收

重构完成后，用户必须能在桌面、平板和移动端完成现有关键流程，并且业务结果含义不变。验收重点不是“看起来像三栏”，而是信息架构、用户路径、状态关系和响应式行为是否清楚。

## 信息架构验收

- 每个路由只有一个主要任务。
- 左侧导航只表示当前工作区的任务入口。
- 主工作区承载当前任务的主要操作和结果。
- 右侧只在选中具体对象时展示详情。
- 全局设置、帮助、历史任务不作为常驻右栏 Tab。
- 有前置依赖的功能使用步骤表达，不使用平级 Tab 伪装。
- 同一对象的不同视角才允许使用 Tab。
- 低频设置默认折叠或进入独立设置页。
- 用户刷新页面后，URL、导航当前态和主内容一致。

## 关键流程验收

### 快速诊断

- 用户可以粘贴章节或使用示例。
- 用户可以选择题材、输入类型和现有必要选项。
- 字数不足时，提示含义与旧结构一致。
- 用户可以启动诊断，并看到运行中状态。
- 本地演示 provider 可以完成完整结果流。
- 共享算力或外部服务失败时，页面展示可读错误摘要，不展示原始 HTML。
- 结果态显示旧结构已有的关键信息：分数、最大问题、证据、读者影响、修改动作、改稿 Prompt、复诊/方法论相关资产。
- 复制 Prompt 可用，复制内容与旧结构一致。
- 重新诊断不丢失当前可编辑输入。
- 结果出现后，输入区不再压过报告主内容。

### 深度质检

- 参考资料、生成评分标准、章节评分按依赖顺序呈现。
- 未生成评分标准时，评分步骤不可直接执行，且提示清楚。
- 已有评分结果能查看总分、问题和证据。
- 证据详情通过检查器或详情页打开。
- `/diagnose/score` 和 `/diagnose/evidence` 旧入口有明确兼容行为。

### 项目工作区

- 当前项目可以选择和查看。
- 快速诊断生成的改稿方案和方法论资产可以在项目页回访。
- 项目切换的状态清理/保留行为与旧结构一致。
- 导出内容不改变字段含义。

### 研究工作区

- 整书任务可以上传或输入文本。
- 用户可以预览章节切分或任务输入。
- 异步任务进度可见。
- 任务结果、图谱、模式和资料库使用清楚的页面或详情视角。
- 历史任务不是所有页面常驻右栏内容。

### 设置工作区

- Provider、模型、Base URL、API Key 等设置可访问。
- 任务页只展示当前模型状态和进入设置的入口。
- 设置变更后回到诊断页，模型状态同步。
- 敏感信息不出现在截图、日志或错误提示中。

## 响应式验收

必须在以下视口验证：

| 视口 | 验收要求 |
| --- | --- |
| 1440×900 | 桌面主工作区是视觉焦点；可选 inspector 不挤压到不可读 |
| 1280×800 | 三块区域如同时出现，主内容仍保留足够宽度 |
| 1024×768 | 左侧可折叠；主任务可完整操作；右侧以 overlay 或隐藏方式处理 |
| 768×1024 | 平板竖屏不依赖横向滚动完成主流程 |
| 390×844 | 单列可完成快速诊断输入、运行、查看结果、复制 Prompt |
| 390×844 + 200% 缩放 | 关键文字、按钮和错误提示不重叠，主 CTA 可访问 |

不得出现：

- 移动端只能看到左栏而无法操作主任务；
- 右栏把主内容挤到不可读；
- Tab、按钮、标题文字溢出容器；
- 横向滚动成为完成主流程的必要条件；
- 关闭详情后用户不知道回到哪里。

## 无障碍验收

- 页面有唯一 `h1`。
- 使用 `header`、`nav`、`main`、`aside` 等 landmark。
- 当前导航项使用 `aria-current="page"`。
- Tab 组件包含 `tablist`、`tab`、`tabpanel`、`aria-controls`，并支持方向键切换。
- 抽屉、检查器和弹窗打开后焦点进入，关闭后焦点返回触发元素。
- Escape 或明确关闭按钮可以关闭临时面板。
- 所有主操作可仅用键盘完成。
- Loading、成功、错误状态有可感知反馈。
- 错误信息是人类可读文本，不包含未清洗 HTML。
- 可调整宽度控件可键盘操作；如果做不到，应替换为明确的宽度选项或固定断点布局。

## 状态和数据验收

- 业务状态与 UI 布局状态分离。
- 清空 UI 布局状态不会删除诊断结果、项目、历史、provider 配置。
- route-scoped UI 状态不会跨路由污染，例如 quick 的 inspector 不会在 settings 中恢复。
- 旧 localStorage key 有兼容读取策略。
- 不改变 API 请求参数含义。
- 不改变缓存命中规则。
- 不改变诊断结果字段含义。
- 不改变历史保存和项目资产生成规则。
- 不引入新的 UI 框架。

## 测试验收

至少需要覆盖：

- route meta 映射；
- workspace 当前导航态；
- 快速诊断空态、输入态、运行态、错误态、结果态；
- 复制 Prompt；
- inspector 打开、关闭、焦点恢复；
- 深度质检步骤依赖；
- provider 设置跨页同步；
- localStorage 兼容读取；
- 错误信息不渲染原始 HTML；
- 响应式 smoke 测试。

建议命令：

```bash
pnpm --filter web test
pnpm --filter web check
pnpm --filter web build
```

完成任何业务页面迁移前，上述命令要么通过，要么在 PR/任务说明中记录明确、可接受的既有失败原因。

## 截图验收

每次迁移一个主要流程后，必须提交截图或录像说明：

- 桌面空态；
- 桌面输入态；
- 桌面运行或错误态；
- 桌面结果态；
- 桌面 inspector 打开态；
- 平板主任务态；
- 移动端主任务态；
- 移动端详情态；
- 200% 缩放关键操作态。

截图必须来自实际运行页面，不使用设计稿替代。

## 回滚验收

- 每个阶段都能只回滚本阶段修改。
- 回滚后旧关键流程可访问。
- 回滚不需要删除用户业务数据。
- 新 UI store 或 layout key 可以安全清空。
- 删除旧结构前，必须确认所有关键路由已经迁移并通过截图和测试验收。

## 完成定义

当以下条件同时满足，才可视为本轮 UX 重构完成：

- 推荐信息架构已经落地到关键路由；
- 快速诊断、深度质检、项目回访、整书研究、设置都能完成主路径；
- 桌面、平板、移动端均通过响应式验收；
- 无障碍验收中的键盘和焦点要求通过；
- 所有必须测试通过或有明确既有失败说明；
- 旧三栏重复层级已删除或被明确标记为回滚期遗留；
- 旧三栏 shell、重复 Tab 和旧布局开关已从主代码路径移除；
- 文档、截图和 Terra 任务状态同步更新。
