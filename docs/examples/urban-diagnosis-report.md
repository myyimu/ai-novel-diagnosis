# 示例诊断报告：都市 AI 初稿 Prompt 约束太泛

这个示例来自 `fixtures/novel-diagnosis/urban-prompt-too-vague.json`，用于展示系统如何提出“问题可能来自正文执行或上一条 Prompt 约束不足”的可核查假设。fixture 只验证工程契约，不验证真实读者效果。

## 示例输入

题材：都市  
稿件来源：AI 生成稿  
上一条 Prompt：

```text
请写一个都市下山高手开局，主角有婚约，对象是集团千金，风格爽一点。
```

节选正文：

```text
前台听见他的来意后愣了一下，随后礼貌地让他稍等。

林越坐在大厅，看见来往白领衣着光鲜，心里并不羡慕。他从小在山里长大，医术、武功都学过一些，只是没有机会施展。

苏总正在开会，婚约的事情可以之后再谈。林越点点头，决定先找地方住下。
```

## 总判断

- Gate：先修改关键问题
- 严重度兼容摘要（quickScore）：5.7/10，不是作品质量分
- 证据/上下文充分度：79%，不是结论正确概率
- 一句话编辑假设：当前开局更像礼貌的设定交代，可能没有兑现作者声明的都市爽文冲突承诺。

核心问题：上一条 Prompt 只给了元素，没有约束冲突强度，导致正文礼貌、平、没有爽点。

## 关键 Issue

### 1. Prompt 只列元素，没有规定冲突和反转

严重程度：高优先级  
问题类型：Prompt 约束

正文证据：

- 第3段：前台听见他的来意后愣了一下，随后礼貌地让他稍等。
- 末段：苏总正在开会，婚约的事情可以之后再谈。

读者影响：读者期待的是被轻视后的当场证明，当前没有压迫，也没有反转爽感。

下一步动作：把下一轮 Prompt 改成必须出现误判、压迫、能力证明和关系变化。

Prompt 约束：第一章必须让主角被误判并在同一场景完成一次小反转。

### 2. 隐藏能力没有展示

严重程度：中优先级  
问题类型：爽点兑现

正文证据：

- 第4段：医术、武功都学过一些，只是没有机会施展。

读者影响：读者只能相信作者声明，得不到爽点兑现。

下一步动作：让主角发现苏家会议室有人突发急症，或一眼看破合作骗局。

Prompt 约束：隐藏能力必须通过行动展示，不能只用旁白说明。

## 作者确认（示例）

- 决定：接受“Prompt 未约束冲突升级”；暂不接受“必须立刻打脸”的单一路径。
- 修改边界：保留婚约、集团前台和主角克制，允许用身份误判或专业能力展示建立反差。
- 注意：Prompt 是否有效必须在保存 V2、检查执行清单并独立复诊后判断。

## 下一轮改稿 Prompt

```text
请重写第一章。保留林越下山履行婚约、苏晚晴是集团千金。

第一场戏必须发生在集团大厅：

1. 前台和助理误判林越是骗子并当众驱赶。
2. 苏家会议室同时出现一个会影响融资的突发危机。
3. 林越用一个低调但可验证的医术/眼力动作解决危机，让苏晚晴第一次改变态度。

禁止只写礼貌等待，禁止用旁白说他很强。
```

为什么这条 Prompt 有用：

- 把模糊爽点变成可检查事件。
- 让能力改变局面。
- 让婚约关系产生推进。

## 复诊检查点

以下是 V2 的检查标准，不代表当前示例已经证明 Prompt 有效：

改完后重新诊断时，至少检查两件事：

- 第一场戏是否出现明确轻视？
- 主角是否在同一场戏改变别人对他的判断？

## 可沉淀方法论

标题：不要只给元素，要给爽点执行标准

可复用规则：给 AI 写第一章时，必须把爽点拆成误判、压迫、行动、反转和关系变化。

自查问题：这条 Prompt 能否检查第一场戏有没有完成反转？
