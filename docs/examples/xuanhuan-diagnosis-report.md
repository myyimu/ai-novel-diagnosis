# 示例诊断报告：玄幻 AI 初稿开局承诺不清

这个示例来自 `fixtures/novel-diagnosis/xuanhuan-ai-draft-opening-promise.json`，用于展示“示例输入 -> 结构化诊断 -> 改稿 Prompt”的最短闭环。

## 示例输入

题材：玄幻  
稿件来源：AI 生成稿  
上一条 Prompt：

```text
请写一个玄幻废柴被逐出宗门的开头，要有雨夜、宗门、旧伤和成长感。
```

节选正文：

```text
执事长老拿出一枚玉简，平静宣布他灵脉残损，三日内离宗。

沈照低着头，没有争辩。

沈照望着远处云雾，忽然觉得胸口那道旧伤微微发热。也许离开这里，反倒能看见更大的天地。
```

## 总判断

- Gate：建议重构这一版
- quickScore：5.0/10
- 置信度：84%
- 一句话诊断：这章像玄幻开场的背景稿，还不像能带动追读的第一章。

核心问题：开局有被逐事件，但没有兑现读者期待的羞辱、代价和反击承诺。

## 关键 Issue

### 1. 被逐事件没有转成读者承诺

严重程度：阻断  
问题类型：市场承诺

正文证据：

- 第2段：执事长老拿出一枚玉简，平静宣布他灵脉残损，三日内离宗。
- 第3段：沈照低着头，没有争辩。

读者影响：读者会觉得主角没有被逼到必须行动的地步，也不知道下一章为什么值得点开。

下一步动作：把放逐改成公开剥夺资格，并让主角当场发现一个能翻盘的证据或机会。

Prompt 约束：第一章前800字必须出现具体羞辱、具体损失和一个可执行反击目标。

### 2. 章末钩子只有情绪，没有具体悬念

严重程度：高优先级  
问题类型：钩子

正文证据：

- 末段：也许离开这里，反倒能看见更大的天地。

读者影响：读者无法判断下一章会发生什么，只能得到泛泛成长预期。

下一步动作：让旧伤发热触发一条明确线索，例如禁地钥匙、仇人印记或被夺灵脉的证据。

Prompt 约束：章末必须写出下一章的具体问题，而不是抒情式展望。

## 下一轮改稿 Prompt

```text
请重写第一章前1500字。主角沈照在宗门演武场被公开逐出，必须出现三件事：

1. 具体羞辱：同门当众指出他灵脉残损并抢走试炼名额。
2. 具体损失：三日内离宗且失去师父遗物。
3. 具体反击目标：旧伤发热指向被夺灵脉的证据，章末让主角决定夜闯藏经阁找证据。

减少回忆和抒情，所有描写都服务压迫、目标和下一章钩子。
```

为什么这条 Prompt 有用：

- 补足读者承诺。
- 把旧伤从氛围改成行动线索。
- 让章末有明确追读问题。

## 复诊检查点

改完后重新诊断时，至少检查两件事：

- 读者能否在前800字知道主角失去了什么？
- 章末是否提出一个下一章必须解决的问题？

## 可沉淀方法论

标题：废柴开局必须先兑现承诺

可复用规则：废柴、被逐、退婚类开头要在前800字给出具体羞辱、具体损失和具体反击目标。

自查问题：读者能否一句话说出主角刚失去了什么、接下来要夺回什么？
