# 示例诊断报告：言情开局设定压过关系钩子

这个示例来自 `fixtures/novel-diagnosis/romance-setting-load.json`，用于展示系统如何提出“设定信息可能压过关系钩子”的编辑假设。fixture 只验证报告结构，不证明模型判断或留存因果。

## 示例输入

题材：言情  
稿件来源：作者正文

节选正文：

```text
那一年，姜家资金链断裂，父亲病倒，母亲远走，妹妹失踪，未婚夫陆迟则在发布会上宣布解除婚约。

姜家别墅灯火通明，宾客名单、礼服品牌、酒店流程、媒体席位都摆在桌上。

她合上文件，对镜子里的自己说，这一次不能再输了。
```

## 总判断

- Gate：先修改关键问题
- 严重度兼容摘要（quickScore）：6.4/10，不是作品质量分
- 证据/上下文充分度：82%，不是结论正确概率
- 一句话编辑假设：如果本章目标是订婚宴关系翻盘，当前背景清单可能推迟核心关系冲突进入场面。

核心问题：设定信息太早铺开，真正的关系冲突和订婚宴钩子没有及时进入场面。

## 关键 Issue

### 1. 上一世信息一次性堆太多

严重程度：高优先级  
问题类型：设定负担

正文证据：

- 第2段：父亲病倒，母亲远走，妹妹失踪，未婚夫陆迟则在发布会上宣布解除婚约。
- 第3段：宾客名单、礼服品牌、酒店流程、媒体席位都摆在桌上。

可能的读者影响：部分目标读者可能尚未建立对男女主关系的兴趣，就被背景信息提高理解成本；如果作者有意营造宿命压迫，则需结合后续场景确认。

下一步动作：只保留今晚会被当众解除婚约这一条核心伤害，其余信息放到后续触发。

Prompt 约束：第一章前1000字最多交代两个过去信息点，必须服务当前场景冲突。

### 2. 女主目标还停在口号

严重程度：中优先级  
问题类型：主角目标

正文证据：

- 末句：这一次不能再输了。

读者影响：读者不知道女主下一步要改哪一件事，也不知道她会如何对抗陆迟。

下一步动作：章末让女主做一个具体选择：提前下楼、换掉证据文件、主动邀请陆迟谈判或当众改流程。

Prompt 约束：章末目标必须是今晚能执行的一步，而不是抽象决心。

## 作者确认（示例）

- 决定：接受“关系冲突进入过晚”；将家族崩塌作为必要压力，但减少一次性交代。
- 修改边界：保留订婚宴、解除婚约和女主重来后的目标，不替作者改变感情立场。
- 注意：真实产品应保存作者意图，避免下一次诊断再次把同一保留项判成错误。

## 下一轮改稿 Prompt

```text
请改写第一章前1200字。

只保留姜宁上一世被陆迟当众解除婚约这一条核心伤害，删除资金链、父母、妹妹、工作室等暂时无关信息。

场景从订婚宴前二十分钟开始，陆迟必须在前800字内出现或发来能改变宴会走向的信息。

章末让姜宁做出一个具体行动：提前下楼并主动改掉原定订婚流程。

重点写关系张力和倒计时，不写背景清单。
```

为什么这条 Prompt 有用：

- 降低首章信息负担。
- 把言情关系钩子前置。
- 让女主目标从口号变成行动。

## 复诊检查点

以下是 V2 的检查标准，不是当前示例已经完成的效果证明：

改完后重新诊断时，至少检查两件事：

- 前1000字是否已经出现关系对抗？
- 章末女主是否做出一个今晚可执行的选择？

## 可沉淀方法论

标题：开局设定只能服务当前场面

可复用规则：第一章前1000字只保留当前场景马上会用到的信息，其余背景等冲突触发后再补。

自查问题：这条背景如果删掉，当前场面是否仍能成立？
